```text
CURRENT
/api/v1/explore
  → post title/content/region substring
  → latest order
  → offset PageResponse

FOUNDATION IMPLEMENTED (not production-wired)
SearchRequest
  → normalized query + filters
  → versioned retrieval
  → SearchCandidateSnapshot
  → deterministic ranking/reranking
  → SearchOutputSnapshot
  → snapshot-bound cursor
  → ephemeral non-persistent result snapshot

INTEGRATION BOUNDARY IMPLEMENTED (disabled, non-persistent)
Legacy response [authority]
  → optional test-only shadow input
  → IP-5 runtime result
  → deterministic comparison evidence
  → legacy response unchanged

SHADOW WIRING CAPABILITY IMPLEMENTED (not production-wired)
Legacy response
  → no-op/default-zero hook contract
  → test-only controlled dispatcher
  → non-persistent structured record
  → response unchanged

CONTROLLED HOOK APPLIED / DEFAULT DISABLED
/api/v1/explore legacy response
  → backend-local disabled bridge
  → identical legacy response returned

TEST/STAGE ACTIVATION IMPLEMENTED (production disabled)
  → explicit search-shadow-test/search-shadow-stage profile + allow gate
  → synthetic in-memory Search Runtime execution
  → bounded non-persistent comparison evidence

FUTURE / NOT IMPLEMENTED
  → production active shadow
  → persisted SearchRun/result snapshot
  → production snapshot-bound cursor
  → search_exposure_v1
```

## 보호 원칙

- Search는 Recommendation ranking/candidate/exposure/metric을 직접 재사용하지 않는다.
- P2 experiment denominator authority는 `recommendation_p2_experiment_exposure`다.
- Data P1/P2 projection은 shadow-only다.
- production application source와 canonical SQL `01..26`은 IP-5에서도 변경하지 않았다.
- Legacy offset page/result는 compatibility evidence로만 읽으며 Search runtime/cursor/exposure authority를 만들지 않는다.

- IP-5 runtime은 in-memory foundation이며 persistence/API/exposure/production cursor authority가 없다.

- IP-6 shadow integration은 기본 비활성이고 legacy response를 변경하지 않으며 persistence/exposure/release/cursor authority가 없다.

- IP-7 wiring은 기본 disabled·sampling 0이며 production Controller/Service 호출부에 삽입되지 않았다.


- IP-8은 production hook을 삽입하지 않았으며 controlled proposal과 disabled-mode equivalence만 완료했다.
- `ip8SearchRegressionClosure`는 IP-1/IP-3..IP-8 및 Recommendation Core 회귀를 묶고 backend Spring/Testcontainers는 별도 명령으로 실행한다.


- IP-9은 Controller return boundary에 controlled void hook을 적용했지만 기본 bean은 완전 no-op이며 production activation/configuration은 없다.
- 외부 Gradle 8.14.5 및 backend Spring/Testcontainers attestation 전 상태는 `IMPLEMENTATION_COMPLETE / EXTERNAL_ATTESTATION_PENDING`이다.

- IP-10은 test/stage explicit activation만 구현했으며 production/default는 disabled, sample 0이다.
- Gradle/JUnit/Spring/PostgreSQL은 사용자 지시로 생략되어 IP-9/IP-10 external attestation은 pending이다.
