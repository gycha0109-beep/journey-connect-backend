# Journey Connect 백엔드 P0~P2 최종 리뷰

## 1. 최종 판정

**조건부 병합 가능**입니다.

P0~P2에서 계획한 기능과 구조는 코드 기준으로 대부분 반영되었습니다. 다만 현재 작업 환경에서는 Gradle 배포 파일을 내려받을 수 없어 전체 테스트를 직접 완료하지 못했습니다. 따라서 새로 추가한 PR CI의 두 작업이 모두 통과해야 병합 가능한 상태로 판정합니다.

필수 PR 체크:

- `Gradle tests (H2)`
- `PostgreSQL 18 / PostGIS migration smoke`

## 3. 잘된 점

- 공개 API와 인증 API의 경계가 비교적 명확합니다.
- 비공개 게시물을 권한 없는 사용자에게 404로 처리해 존재 여부 노출을 줄였습니다.
- 목록 조회 시 작성자·지역 fetch와 반응 수 일괄 집계를 사용해 N+1을 억제했습니다.
- 좋아요·북마크는 애플리케이션 사전 확인과 DB 유니크 제약을 함께 사용합니다.
- 크루 정원 판정은 크루 행 비관적 잠금으로 직렬화했습니다.
- 피드는 `createdAt + id` 복합 커서를 사용해 같은 생성 시각의 게시물도 안정적으로 페이지 이동합니다.
- 리프레시 토큰 원문을 DB에 저장하지 않고 해시만 저장하며, 재발급 시 회전합니다.
- Flyway와 Hibernate 검증의 책임을 분리했습니다.
- PR CI에 H2 테스트와 실제 PostgreSQL/PostGIS 기동 검사를 분리했습니다.

## 4. 병합 전 확인이 필요한 항목

### P0 — 반드시 확인

#### 4.1 최초 CI 실행 결과

현재 환경에서는 `services.gradle.org` DNS 접근 실패로 `./gradlew clean test`를 완료하지 못했습니다. 정적 검토만으로 컴파일·Spring Context·Repository 쿼리 파싱 성공을 확정할 수 없습니다.

조치:

- PR에서 CI 두 작업이 모두 성공한 뒤 병합
- 실패하면 전체 Gradle 로그와 `application-startup.log`를 기준으로 수정

#### 4.2 기존 DB의 빈 지역명 마이그레이션

`V2__p2_domain_extensions.sql`은 공백이 아닌 기존 `region_name`을 LEGACY 지역으로 이관합니다. 기존 데이터에 빈 문자열 또는 공백만 있는 지역명이 있으면 `region_id`가 null로 남을 수 있고, 조건부 `NOT NULL` 적용이 생략됩니다.

위험:

- 엔티티는 `region_id`를 필수로 가정하지만 DB는 nullable로 남을 수 있음
- Hibernate 스키마 검증 실패 또는 런타임 정합성 오류 가능

권장 보완:

- 마이그레이션 전에 빈 지역명 존재 시 명시적으로 실패시키기
- 또는 `UNKNOWN` 지역으로 이관한 뒤 `region_id NOT NULL`을 무조건 적용하기

#### 4.3 `baseline-on-migrate` 운영 적용 범위

기존 로컬 DB 이관을 위해 전역으로 `baseline-on-migrate: true`가 설정되어 있습니다. 잘못 연결된 비어 있지 않은 DB도 기준선 처리될 수 있으므로 운영 환경에서는 위험합니다.

권장 보완:

- 기존 DB 전환 완료 후 기본값을 false로 변경
- 필요한 로컬/전환 프로필에서만 true 적용

## 5. 우선 보완 권장 사항

### P1 — 병합 직후 우선 처리

#### 5.1 크루 승인과 참가 취소 간 동시성

승인은 크루 행을 잠그지만 `cancelJoin`은 같은 잠금을 사용하지 않습니다. 승인과 취소가 동시에 실행되면 마지막 flush 결과에 따라 상태가 덮일 수 있습니다.

권장 보완:

- 취소도 크루 또는 신청 행을 `PESSIMISTIC_WRITE`로 조회
- 또는 `CrewMember`에 `@Version`을 추가하고 충돌을 409로 처리
- 승인/취소 동시 요청 통합 테스트 추가

#### 5.2 페이지 크기·정렬 화이트리스트

`Pageable` 기반 API는 클라이언트가 큰 `size`나 예상하지 않은 정렬 필드를 전달할 수 있습니다.

권장 보완:

- `spring.data.web.pageable.max-page-size` 설정
- API별 허용 정렬 필드 제한
- 피드 외 목록도 필요 시 커서 방식 검토

#### 5.3 리프레시 토큰 정리 정책

만료 토큰 삭제 Repository는 있지만 실제 호출 경로 또는 스케줄러가 없습니다. 장기 운영 시 테이블이 계속 증가합니다.

권장 보완:

- 주기적 만료·폐기 토큰 삭제
- 사용자별 활성 세션 수 제한
- 토큰 계열 또는 세션 식별자 도입 검토

#### 5.4 PATCH 계약 명확화

프로필의 `bio/profileImageUrl`은 null 전달 시 제거되고, 게시물의 빈 제목·본문은 변경 없이 무시됩니다. 필드별 null·빈 문자열 의미가 일관되지 않습니다.

권장 보완:

- API 명세에 미전송/null/빈 문자열 동작 명시
- 필수 문자열의 빈 값은 400으로 통일

### P2 — 기능 확장 시 처리

#### 5.5 PostGIS 인덱스와 실제 검색식 정합성

현재 인덱스는 geometry 컬럼에 생성되지만 검색 쿼리는 `center::geography`로 캐스팅합니다. geography 캐스팅 검색에서 기존 geometry GiST 인덱스를 충분히 활용하지 못할 수 있습니다.

권장 보완:

- `USING GIST ((center::geography))` 표현식 인덱스 추가
- 또는 컬럼 자체를 geography로 설계
- 실제 `EXPLAIN ANALYZE`로 인덱스 사용 확인

#### 5.6 지도 기능의 현재 범위

현재 PostGIS 모델은 **지역 중심점과 주변 지역 검색**입니다. 게시물·장소 개별 좌표, 지도 영역 검색, 거리순 게시물 검색은 아직 구현되지 않았습니다.

향후 결정할 항목:

- 좌표를 게시물, 장소, 일정 중 어디에 저장할지
- geometry/geography 선택
- 지도 viewport 검색 또는 반경 검색 계약
- 주소·좌표 변환 책임과 지도 공급자

#### 5.7 크루 운영 API

현재 생성·조회·신청·승인은 가능하지만 크루 수정, 모집 종료, 삭제, 승인 멤버 목록 API는 없습니다. 실제 크루 화면 요구사항이 확정되면 추가해야 합니다.

#### 5.8 운영용 상태 확인 API

`/api/v1/test/**`가 메인 소스와 공개 보안 경로에 포함되어 있습니다.

권장 보완:

- 테스트 컨트롤러를 local/test 프로필로 제한
- 운영 상태 확인은 Actuator health 등 별도 엔드포인트로 전환
- 운영에서 Swagger 공개 여부도 별도 정책 적용
